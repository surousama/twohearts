# 闪避后移动动画资源切换

## Goal

作为第二章基础战斗模块在“最小预输入已跑通”之后的一个局部收口子 task，从主程序视角补齐闪避后动作衔接的稳定承载方案：为 Dodge 新增一套“闪避后进入移动”的动画资源配置入口，并在闪避结束时根据玩家是否仍有移动输入，切换到这套专用资产，消除当前“先起身再移动”的僵硬衔接。

## What I already know

* 当前最小预输入等基础建设已经跑通，说明 Dodge、公共动作上下文、输入评估与后续输入承接主链路已具备继续叠加局部手感收口的条件
* 当前闪避动画实际上区分两套后续表现：
  闪避后原地收势；
  闪避后直接进入移动
* 当前运行中只接了“闪避后原地收势”这套资产，因此玩家在持续推移动输入时，角色仍会先播起身动作，再切到移动，表现非常僵硬
* 当前 Dodge 主链路仍是 `UE5 + GAS + C++`，方向选择与资源入口已集中在 `FTwoHeartsDodgeConfig` / `UTwoHeartsGA_Dodge` / `AtwoheartsCharacter`，不应为了这次表现问题回退到 Character 级临时状态机
* 角色侧已经缓存 `CachedMoveInput`，并且当前 Dodge 方向解析、Montage 选择与资源校验都围绕该链路展开，具备在闪避结束时复用“是否仍有移动意图”这一判断口径
* 上游 `05-19-dodge-second-pass-polish` 已确认当前 Dodge 方向口径为“保留角色朝向不变的相对方向闪避”，本轮不应破坏该结论

## Requirements

* 从主程序视角确认：本问题适合继续叠加在现有 Dodge 架构上解决，方案边界收敛为“新增配置入口 + 闪避结束时的移动意图分流 + 表现切换”，不重做 Dodge 主状态流
* 在现有 `FTwoHeartsDodgeConfig` 上新增一套与当前 Dodge 后续表现相对应的“闪避后移动”资源配置入口；若当前方向资源是按八方向配置，本轮新增入口也应保持同一粒度，而不是临时塞成单一全局资产
* 运行时在“闪避动作结束并准备衔接后续表现”的明确时机检测玩家是否仍有有效移动输入；若存在，则选择“闪避后移动”资产，否则保持当前原地收势资产
* 该判断应优先复用现有输入缓存与 Dodge 生命周期口径，不新增与公共动作上下文冲突的私有状态机，不把判断分散到多个互相覆盖的蓝图分支
* 需要明确本轮的切换时机归属：
  应发生在 Dodge 结束衔接阶段；
  不应改成 Dodge 启动时预判整段后续；
  不应影响 Dodge 本体的方向解析、Root Motion、无敌帧、冷却与打断链路
* 若这次切换依赖编辑器资源挂载、Montage 段落、Notify 或动画蓝图出口，必须把实际配置入口写清，避免实现者只改 `C++` 却漏掉资产承接位置
* 若当前最小预输入会导致闪避结束前后存在“输入被消费但移动 intent 仍应成立”的窗口，本轮应采用对玩家体感正确的判断口径，并在实现记录中写清

## Acceptance Criteria

* [ ] `DodgeConfig` 或等价正式配置入口中，已存在一套清晰可挂载的“闪避后移动”资产位置，命名与原有 Dodge 资源结构一致
* [ ] 玩家在闪避结束前后持续保持移动输入时，角色不再先播原地起身再进入移动，而是直接走“闪避后移动”衔接表现
* [ ] 玩家在闪避结束时没有移动输入时，角色仍保持当前原地收势表现，不回归异常切错
* [ ] 本轮没有破坏现有 Dodge 的方向解析、Root Motion、无敌帧、冷却、打断、最小预输入承接和公共动作上下文链路
* [ ] 运行时判断口径足够稳定，能覆盖至少：
  持续按住移动后闪避；
  闪避过程中继续推移动；
  闪避结束瞬间松开移动
* [ ] 若切换失败，开发者能够从代码落点、配置入口或调试输出快速定位是“未配置移动后资产”还是“结束时未识别到移动输入”

## Out of Scope

* 重做 Dodge 整体架构、方向算法或角色朝向策略
* 重做最小预输入、公共战斗语义层或普攻链路
* 一次性扩成完整的“所有动作后续衔接状态机”
* 新增格挡、受击、伤害或联机同步改造
* 为了这次需求回退到 Character 蓝图中堆叠临时分支，绕开当前 `GAS + C++` 正式承载

## Technical Notes

* 角色判定：主程序视角下已确认可在现有架构上稳定叠加，交付目标应写到资深程序可直接实施
* 父 task：`05-19-chapter2-basic-combat`
* 当前任务与项目主顺序关系：属于第二章基础战斗模块在“最小预输入已跑通”后的局部手感收口，不改变主顺序，也不提前开启新阶段
* 建议优先代码落点：
  `Source/twohearts/twoheartsCharacter.h`
  `Source/twohearts/twoheartsCharacter.cpp`
  `Source/twohearts/TwoHearts/Combat/Gameplay/Abilities/TwoHeartsGA_Dodge.h`
  `Source/twohearts/TwoHearts/Combat/Gameplay/Abilities/TwoHeartsGA_Dodge.cpp`
* 建议优先资源/配置落点：
  `Content/ThirdPerson/Blueprints/BP_ThirdPersonCharacter.uasset`
  当前已挂载 Dodge 相关 Montage 的配置入口
  现有 Dodge Montage 与其结束 Notify / 段落配置
* 当前实现者开工前应先确认：
  1. “闪避后移动”资产是否也是按方向区分，还是已有统一资源命名规则但尚未接入口
  2. 结束时移动输入判断究竟以 `CachedMoveInput`、角色速度，还是两者组合为准；默认优先以输入 intent 为准，避免体感滞后
  3. 切换逻辑更适合落在 `GA_Dodge` 结束前决策，还是角色/动画承载层的正式接口中统一决策；无论选哪处，都只能保留一个真相源
* 建议实施顺序：
  1. 梳理当前 Dodge 结束后原地收势资产的正式配置与播放入口
  2. 新增“闪避后移动”配置字段，并完成编辑器可挂载入口
  3. 在 Dodge 结束衔接点接入移动意图判断与资产选择
  4. 本地验证三类关键输入时序，并记录已知限制
