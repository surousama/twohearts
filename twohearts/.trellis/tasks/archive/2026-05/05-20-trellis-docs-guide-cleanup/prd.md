# Trellis 文档指南清理与统一

## Goal

统一双心印项目中 Trellis 使用说明的入口文档、路径写法和 `docs/` 目录维护口径，使后续在公司与家两套环境下都能直接点击跳转并按同一规则使用。

## Requirements

* 将 `g:\twohearts\docs\aTrellis关键位置导航.md` 改名为 `aTrellis使用指南.md`
* 将该文档中的绝对路径改为相对路径，并保持 Markdown 链接可直接点击跳转
* 在该文档中补充 Trellis 常用操作说明，包括开发者初始化、task 操作、上下文查询、会话记录、常见排查
* 统一项目文档中的 `docs/` 维护口径：
  `docs/` 仍会维护，但主要保留策划类文档；
  开发任务文档、测试文档与运行过程文档不再作为持续维护主区
* 将能确认的绝对路径引用统一改为相对路径

## Acceptance Criteria

* [ ] 新文档名为 `aTrellis使用指南.md`
* [ ] 新文档中的核心入口都可点击跳转
* [ ] 新文档包含常用命令与初始化步骤说明
* [ ] `.trellis/` 与 `docs/` 中相关文档的 `docs/` 维护口径已统一
* [ ] 已知绝对路径引用已改为相对路径

## Out of Scope

* 调整 Trellis 脚本行为本身
* 新增或删除现有业务代码
