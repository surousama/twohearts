# 普通攻击武器配置与持武器切换

## Goal

为当前角色补齐普通攻击所需的武器配置，并完成移动状态与普通状态之间的持武器切换，让普通攻击表现不再是空手，同时保持当前第二章基础战斗阶段内可接受的轻量实现边界。

## What I already know

* 用户目标是新建一个 task，处理“普通攻击没有武器”的问题。
* 需求包含两部分：补一个可用于普通攻击的武器；完成“移动时不持有武器，普通时持有武器”的切换。
* 当前项目仍处于第二章“基础战斗模块”阶段，主线顺序仍强调先完成基础闪避收口，再推进更高层公共语义。
* 当前普通攻击已经正式承载在 `UE5 + GAS + C++` 路线上，普攻 `1-2-3` 已由 `Gameplay Ability` 驱动。
* 当前任务更像单轮可验收的战斗表现/资源配置子 task，已挂到父 task `05-19-chapter2-basic-combat` 下。

## Requirements

* 为当前角色新增一套可用于普通攻击表现的武器配置。
* 角色在移动状态下不持有武器，在普通状态下持有武器。
* 这里的“普通状态”按当前确认口径，覆盖静止待机与普通攻击状态；一旦进入移动状态则切为不持有武器。
* 本轮先直接复用仓库内已有武器资源完成第一版，不等待新资源制作。
* 切换方案应与当前普通攻击正式承载路线兼容，不能回退到 Character 级临时战斗状态机。
* 若实现中存在过渡方案，必须明确标注为当前阶段可接受的过渡实现。

## Acceptance Criteria

* [ ] 当前角色普通攻击播放时不再表现为空手。
* [ ] 角色进入移动状态后，武器会切换为不持有状态。
* [ ] 角色回到普通状态后，武器会切回持有状态。
* [ ] 本轮方案不引入与后续公共战斗语义层冲突的新动作规则。

## Out of Scope

* 公共战斗语义层设计与实现。
* 预输入、受击、伤害、格挡等后续阶段功能。
* 为未来所有武器类型一次性抽象完整武器系统。
* 进入战斗、退出战斗时的完整武器切换规则与状态设计。
* 回退到旧的 Character 级普攻承载方式。

## Technical Notes

* 当前角色判定：主程序视角。
* 父 task：`05-19-chapter2-basic-combat`
* 当前 task：`05-23-normal-attack-weapon-switch`
* 当前阶段关系：属于第二章基础战斗模块中的局部表现与配置补齐，不改变主线阶段顺序。
* 已确认边界：静止待机和普通攻击时持武器；开始移动时切为不持武器。
* 当前代码中尚无现成的武器组件、装备槽位或挂点逻辑，本轮预计需要补一个最小武器承载方案。
* 当前武器配置目标是为后续伤害等功能预留最小承载，不在本轮扩展成完整武器系统。
* 当前仓库内已存在可复用的剑资源，例如 `Content/pack/StaticMesh/SM_LongSword.uasset`、`Content/pack/StaticMesh/SM_ShortSword.uasset`、`Content/Chinese_Warrior/Mesh/SM_Chinese_Hanfu_Dress_Warrior_Sword.uasset`。
