# 公共战斗语义层开工评估与实施拆分

## Goal

作为第二章基础战斗模块下的主程序评估子 task，在基础闪避阶段验收通过后，复核当前是否已经具备进入“公共战斗语义层”阶段的条件，并输出可直接派给资深程序的实施拆分。

## What I already know

* 当前第二章正确顺序仍是：基础闪避正式落地收口 -> 公共战斗语义层 -> 最小预输入
* 公共战斗语义层不应建立在“普攻 + 验证型 Dodge”之上，而应建立在“普攻 + 已稳定基础闪避”之上
* 旧文档中已经存在公共战斗语义层的主程序实现方案草案
* 本 task 不是直接编码实现，而是主程序视角的阶段 gate 评估与实施拆分

## Requirements

* 以前置子 task 已给出基础闪避阶段验收结论为前提
* 复核普攻与基础闪避之间哪些共享概念已经稳定，哪些仍然是过渡实现
* 判断当前是否适合进入公共战斗语义层；若不适合，必须明确阻塞项
* 若适合开工，需要输出首轮实施模块拆分、建议顺序和可直接验收的子 task 粒度
* 结论必须服务于下一轮真实推进，不能只写成大而全背景报告

## Acceptance Criteria

* [ ] 明确给出“现在是否允许进入公共战斗语义层”的主程序结论
* [ ] 若允许进入，已拆出首轮可执行子 task 或明确实施顺序
* [ ] 若暂不允许进入，已写清阻塞项和回退入口
* [ ] 输出结果能够直接作为后续资深程序实施或继续拆单的上游依据

## Out of Scope

* 公共战斗语义层实际代码实现
* 最小预输入实现
* 受击、伤害、格挡联动实现
* 仅凭预期直接跳过基础闪避阶段验收
