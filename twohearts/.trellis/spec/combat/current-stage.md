# 第二章当前阶段口径

## 当前有效结论

1. 第二章基础战斗模块已经完成第一轮正式战斗底座搭建
2. 普通攻击正式承载稳定落在 `UE5 + GAS + C++`
3. `AtwoheartsCharacter` 当前只应继续承担 `ASC` 持有与初始化、输入转发、角色资源配置和调试承载
4. 普攻 `1 / 2 / 3` 已形成单路径 `Gameplay Ability` 承载
5. 普攻阶段语义 `Startup / Active / Recovery / LogicEnded` 已建立
6. 基础闪避、公共战斗语义层、Guard 规则与预输入二期都已完成一轮正式落地
7. 玩家侧已具备最小正式的命中 / 伤害 / 受击状态链路
8. 当前阶段允许局部过渡实现，但必须明确标注为过渡，不可伪装成长期正式结构
9. 当前第二章的主要缺口已从“动作底座未建完”转为“双向攻防闭环与正式表现层未补齐”

## 当前正确顺序

1. 普通攻击正式命中派发基础
2. 敌方受击与生命最小闭环
3. 玩家受击表现正式化
4. 不可格挡与 GuardBreak 规则实现
5. 完整格挡表现实现
6. 战斗属性真相源收口
7. 攻防样本扩展与回归验收

## 顺序判断原则

1. 当前第一优先级不再是重做动作底座，而是补齐“我打敌”的正式闭环
2. 完整 Guard 表现不应早于不可格挡 / GuardBreak 规则稳定，否则表现输入会持续变化
3. 战斗状态真相源收口不宜早于双向攻防与关键表现闭环，但也不应无限后拖
4. 若建议改变当前顺序，必须明确说明收益、风险和为什么当前顺序已不成立

## 对 AI 的要求

1. 当前没有明确指示时，不要主动跨越顺序
2. 若当前功能依赖尚未完成的底座，应先指出阻塞
3. 若任务属于“过渡实现”，应明确标注为当前阶段可接受的过渡实现
4. 不要为了单个子功能方便，提前把多个后续系统的公共底座做重
5. 不要再沿用“Guard 还没开始”“预输入还没做”的旧阶段判断
