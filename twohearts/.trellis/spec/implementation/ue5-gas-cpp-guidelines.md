# UE5 + GAS + C++ 实现路线

## 核心路线

1. 双心印当前战斗与技能系统继续沿用 `UE5 + GAS + C++`
2. 能力正式承载优先落在 `Gameplay Ability`
3. 角色基类不承载越来越多的战斗状态机

## 承载原则

1. 长期正式承载优先放在 `Ability`、公共语义层和相关 `C++` 组件
2. `AtwoheartsCharacter` 只保留角色级入口、输入桥接、配置入口和调试承载
3. 后续闪避、格挡、受击、预输入也应延续 Ability 与公共语义层扩展，而不是回退旧方案

## 禁止事项

1. 不轻易跳出既定技术路线
2. 不为了局部方便直接写 Character 级临时战斗状态机
3. 不提前把未来多个系统的公共底座做得过重
