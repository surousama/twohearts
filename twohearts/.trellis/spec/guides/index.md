# 双心印思考指南

## 用途

这一层不是项目规则本体，而是帮助 AI 在开工前补齐“容易忽略但会造成返工”的思考维度。

## 可用指南

| Guide | Purpose | When to Use |
|-------|---------|-------------|
| [Code Reuse Thinking Guide](./code-reuse-thinking-guide.md) | 识别重复模式，减少重复实现 | 看到类似逻辑散落在多个模块时 |
| [Cross-Layer Thinking Guide](./cross-layer-thinking-guide.md) | 检查跨层数据流与职责边界 | 任务跨越输入、Ability、角色、表现层时 |
| [Stage Order Thinking Guide](./stage-order-thinking-guide.md) | 判断当前是否走对开发顺序 | 想提前叠功能或补底座时 |

## 双心印额外提醒

1. 当前很多返工来自“顺序不对”，而不是“代码不会写”
2. 当前很多技术债来自“为局部功能偷偷造一套新规则”
3. 当前很多联调问题来自“蓝图与 C++ 真相源分裂”
