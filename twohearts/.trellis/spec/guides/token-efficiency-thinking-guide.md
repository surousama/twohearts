# 低 token 工作思考指南

## 什么时候必须读这份指南

1. 你发现当前任务主要成本不是写代码，而是读流程、搜代码和反复试命令
2. 你准备在大目录下直接全文搜索或整文件通读
3. 你准备在 Windows PowerShell 5.1 下跑一串不确定是否兼容的命令

## 检查问题

1. 当前要找的是“候选文件”还是“最终实现细节”？若只是找落点，是否应先 `rg --files`
2. 当前必须读整份文档 / 整个 `.cpp` 吗，还是只需要命中片段、索引或某个函数
3. 这一步是独立读取，还是依赖前一步产物？若存在依赖，是否应避免并行
4. 当前环境里有没有已知坑位可直接绕开，例如 PowerShell 5.1 的执行策略与 `&&`
5. 这次优化是否仍保留了 spec、PRD、验证与收尾，而不是靠“少做一步”来省 token

## 双心印当前推荐做法

1. 启动阶段先读 `trellis-start`、`get_context.py`、相关 `index.md`，不要反复全文读取 `workflow.md`
2. 代码搜索优先“两段式”：
   先收路径；
   再搜内容；
   最后按命中片段局部读取
3. 大文件优先读函数周边或 `Select-Object -Skip N -First M` 片段，不默认整文件展开
4. 写命令时优先一次成功，而不是先试高风险写法再返工；PowerShell 5.1 下不要用 `&&`
5. 若经验适合长期复用，应优先回写 `.trellis/spec/`，不要把省 token 规则继续堆到顶层全局提示里
