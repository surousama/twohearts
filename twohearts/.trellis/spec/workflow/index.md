# 双心印工作流规范索引

## 用途

这一层定义 AI 在双心印项目中的标准工作方式。它优先级高于通用编程直觉，目的是让 AI 贴合当前真实协作流程，而不是套用默认软件项目节奏。

## 开工前必读

1. 先读 [conversation-rules.md](./conversation-rules.md)
2. 再读 [role-routing.md](./role-routing.md)
3. 若任务涉及迁移期文档维护，再读 [migration-rules.md](./migration-rules.md)

## 质量检查

1. 开工前必须先判定当前角色
2. 阅读时只读当前任务直接需要的内容，不默认扩读整个项目
3. 需要写代码时，必须已经获得明确指示
4. 迁移期内不得删除旧 `docs` 文档
