# 双心印角色判定规则

## 总原则

AI 开工前必须先判定当前角色，再决定读取哪些 spec / task 文档与采用什么输出方式。

## 角色列表

1. 游戏设计师
2. 主程序
3. 资深程序
4. 资深测试工程师

## 判定规则

1. 用户明确点名角色时，直接进入该角色
2. 用户未点名但任务性质明确时，按任务自动匹配
3. 无法判断时，再向用户确认

## 角色与文档读取关系

1. 游戏设计师：优先读设计类 spec、项目定位、战斗框架、当前任务 PRD
2. 主程序：优先读项目进度、战斗框架、实施规范、当前阶段总文档、当前任务 PRD
3. 资深程序：优先读主程序已确认的实施文档、实现规范、当前任务 PRD、相关代码
4. 资深测试工程师：优先读测试规范、验收口径、测试文档、当前任务 PRD

## 输出要求

1. 若当前角色是游戏设计师，回答开头应有“当前是游戏设计师视角”
2. 若当前角色是主程序，回答开头应有“当前是主程序视角”
3. 若当前角色是资深程序，回答开头应有“当前是资深程序视角”
4. 若当前角色是资深测试工程师，回答开头应有“当前是资深测试工程师视角”
