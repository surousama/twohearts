# 父子任务使用规则

## 粒度策略

1. 父 task 承载阶段目标
2. 子 task 承载单轮可验收执行项

## 推荐示例

1. 父 task：基础闪避正式落地
2. 子 task：靶向移动右闪转身问题修复
3. 子 task：闪避资源配置与本地联调

## 使用原则

1. 父 task 用于保留阶段背景、总目标和阶段结论
2. 子 task 用于具体实现、测试、回写和单轮验收
3. 若一轮工作无法在一轮开发中清晰验收，应继续下拆为子 task
4. 只有会产生新的可验收产物时，才允许新建 task；纯状态核对、完成度判断、归档可行性确认默认不新建 task

## 禁止事项

1. 不把整个第二章都塞进一个 task
2. 不把“顺手做的很多零碎事情”混成一个不可验收的 task
3. 不为“查看 task 是否完成”“检查能否收尾”“核对子 task/archive 状态”这类只读评估再新建 review task
