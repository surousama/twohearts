# 双心印 PRD 编写规则

## 必备内容

1. 目标
2. 当前已知信息
3. 需求
4. 验收标准
5. 明确不做的范围
6. 技术备注

## 双心印特有要求

1. 必须写清当前阶段目标
2. 必须写清当前明确不做的范围
3. 必须说明当前任务与项目主顺序的关系
4. 若任务来自旧 `docs`，应写明来源文档，方便迁移回溯
5. `prd.md` 默认应面向实际执行者书写，能让资深程序 / 资深测试直接开工；不要把 `prd.md` 写成只服务“再拆 task”的中间说明
6. `prd.md` 正文默认使用中文；只有 Trellis、Codex、JSONL、PRD、C++、Blueprint、PIE、HUD 等必要专有名词可保留英文

## 推荐风格

1. 简洁
2. 可直接派活
3. 可直接验收

## 禁止事项

1. 不写成大而全背景报告
2. 不把长期规范全部塞进 `prd.md`
3. 不遗漏验收口径
4. 不把 task 写成“为了管理 task、拆分 task、设计 task 而存在”的中间文档
5. 不把整段需求、备注、检查说明写成英文；除必要专有名词外，默认中文化
