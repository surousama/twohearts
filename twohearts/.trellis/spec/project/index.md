# 双心印项目规范索引

## 用途

这一层用于说明《双心印》是什么项目、当前推进到哪里，以及 AI 在进入任何任务前必须先理解的项目级事实。

## 开工前必读

1. 若当前任务涉及项目定位、玩法目标、双人共斗卖点，先读 [product-positioning.md](./product-positioning.md)
2. 若当前任务涉及当前开发阶段、下一步顺序、是否适合继续叠功能，先读 [current-progress.md](./current-progress.md)
3. 若当前任务只涉及局部实现，至少也要先确认当前阶段不与主线顺序冲突

## 质量检查

1. 不要把双心印误判成通用 ARPG 项目，必须保留双人共斗、结印、合印这三层核心特色
2. 不要忽略当前仍处于第二章基础战斗模块推进阶段的事实
3. 如果输出结论会改变开发顺序，必须与当前进度口径对齐，而不是凭空换路线
