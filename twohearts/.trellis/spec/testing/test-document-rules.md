# 测试文档维护边界

## 可改范围

1. 测试计划
2. 测试用例
3. 测试报告
4. 问题记录

## 不可改范围

1. 设计文档
2. 技术评估文档
3. 技术实现文档
4. 业务代码

## 输出风格

1. 简洁
2. 可复用
3. 以验收和问题追踪为中心

## 迁移说明

1. `docs/测试文档/` 目录中的旧测试文档继续保留用于回溯
2. 后续持续维护的测试过程、问题记录和验收产物，优先写入 `.trellis/tasks/` 与相关 Trellis 文档
3. `docs/` 目录主要继续维护策划类与长期背景类文档，不再作为测试文档持续维护主区

由旧文档 `../docs/a资深测试工程师职责.md` 提炼。
