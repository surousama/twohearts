# 整体技术结论

1. 总体技术栈
   沿用 `UE5 + GAS + C++`，不跳出当前项目路线。
2. 总体落地原则
   核心判定、状态流转、输入缓存、命中与伤害统一放在 C++；
   动画资源、Montage、通知点、特效与表现层留给蓝图和数据配置。
3. 当前状态判断
   最小普通攻击闭环已经完成，并已完成从 Character 级过渡状态向正式 GAS 承载的迁移；
   当前普通攻击已进入正式战斗架构的第一版落地阶段。
4. 当前主程序结论
   AbilitySystem 基础底座已经完成并形成历史留档；
   普通攻击正式单路径 GAS 承载已经完成第一版落地；
   下一步不建议直接先做预输入；
   当前应先进入“普通攻击阶段标记与基础打断”；
   在此基础上再做最小预输入和后续战斗模块。

# 当前代码状态与注意点

1. 当前普攻最小闭环没有发现阻断本地验收的严重漏洞，可视为开发中的稳定过渡版本。
2. 当前已完成的关键承载切换
   普攻输入、段序进入、Montage 播放、段间衔接、连段结束清理和主要调试状态，已经迁入 Gameplay Ability；
   `AtwoheartsCharacter` 当前只保留 `ASC`、输入转发、角色侧配置和调试承载职责。
3. 当前阶段确认下来的技术选型
   普通攻击长期按“一段普攻一个 Ability”组织；
   当前阶段采用“角色持有配置，Ability 持有逻辑”的最小分工；
   下一段激活当前按独立 Ability Tag 驱动；
   当前段结束改为依赖 `UAbilityTask_PlayMontageAndWait` 的完成/打断回调，而不是 Character Timer。
4. 当前必须记住的后续改造点
   当前段落切换虽然已迁入 Ability，但仍以 Montage 完成回调为主；
   后续进入正式阶段语义后，再统一改为动画通知或动作逻辑事件驱动；
   当前还没有正式 `Startup / Active / Recovery / LogicEnded` 公共语义。
5. 当前不宜直接叠加的内容
   不建议在当前普攻 Ability 第一版落地上直接混入完整预输入、受击、伤害、闪避、格挡联动；
   应按既定顺序先补阶段标记与基础打断，再继续向后扩展。
6. 当前程序侧最小共识
   普通攻击长期按 Ability 组织；
   阶段边界以后以动画通知或动作逻辑事件为主；
   打断关系按“动作类型 + 当前阶段 + 新输入类型”判定；
   通用语义层不是当前开工项，而是后置收束项。
