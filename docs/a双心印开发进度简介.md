# a双心印开发进度简介

1. 已经完成角色基础移动。
2. 已经完成 GAS（Gameplay Ability System）基础底座接入。
3. 普通攻击已经完成从 Character 级过渡逻辑到正式 `AbilitySystem` 承载的第一轮迁移。
4. 当前普通攻击正式采用 GAS 单路径承载，`1-2-3` 三段普攻已由 `Gameplay Ability` 驱动。
5. 普攻阶段语义 `Startup / Active / Recovery / LogicEnded` 已建立，并已接入统一公共动作上下文。
6. 基础闪避已经完成第一轮正式化：方向解析、Montage 驱动、Root Motion 校验、冷却、无敌帧、完成链路和调试口径均已接通。
7. 公共战斗语义层已经完成第一轮正式落地，当前已存在：
   公共动作上下文；
   统一动作类型 / 阶段 / 结束原因；
   普攻与 Dodge 的统一动作状态承载；
   普攻被 Dodge 打断的公共接入口；
   统一输入评估结果口径 `ExecuteNow / BufferInput / Reject`。
8. 当前已经具备“最小预输入接入口”，但还**没有**完整预输入系统：
   现在已经能统一判断输入是立即执行、缓冲还是拒绝；
   但还没有正式的缓冲队列、统一消费时机和完整预输入执行机制。
9. 当前技术路线已明确沿用 `UE5 + GAS + C++`，并按“核心逻辑在 C++、表现层与资源配置在蓝图/数据”推进。
10. 当前主线顺序已经推进到：
    最小预输入
    -> 受击与伤害
    -> 格挡
    -> 更完整的战斗公共规则收束。
11. 当前**还不建议**直接进入格挡正式开发。原因不是公共语义层没做好，而是：
    完整预输入尚未完成；
    受击与伤害系统尚未正式落地；
    还没有可稳定验证攻防链路的最小敌人系统。
12. 当前更合理的下一步是：
    先继续收口普攻与 Dodge 的基础手感；
    再完成最小预输入；
    然后补最小受击与伤害；
    再补最小敌人系统；
    最后进入格挡。
13. 武器、伤害系统、敌人系统与更完整手感调优，不建议现在一口气同时补齐：
    角色基础手感应持续边开发边收口；
    伤害与受击应排在格挡之前；
    敌人系统建议在最小伤害链路建立后尽快补一个可验证版本；
    武器正式承载建议放在玩家基础攻防闭环跑通后补，避免过早叠加表现层复杂度。
14. 项目当前仍在持续梳理设计文档、技术文档和职责文档，并逐步把阶段性稳定结论回收进 Trellis spec 与总览文档。
