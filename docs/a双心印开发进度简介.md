# a双心印开发进度简介

1. 已经完成角色基础移动。
2. 已经完成 `GAS`（Gameplay Ability System，应用能力系统）基础底座接入。
3. 普通攻击已经完成从 Character 级过渡逻辑到正式 `AbilitySystem` 承载的第一轮迁移，`1-2-3` 三段普攻由 `Gameplay Ability` 正式驱动。
4. 普攻阶段语义 `Startup / Active / Recovery / LogicEnded` 已建立，并已接入统一公共动作上下文。
5. 基础闪避已经完成第一轮正式化：方向解析、`Montage`、`Root Motion`、冷却、无敌帧、完成链路和调试口径均已接通。
6. 公共战斗语义层已经完成第一轮正式落地，当前已存在：
   公共动作上下文；
   统一动作类型 / 阶段 / 结束原因；
   普攻与 Dodge 的统一动作状态承载；
   统一输入评估结果口径 `ExecuteNow / BufferInput / Reject`。
7. 最小预输入已经完成首轮正式闭环，并做过白盒验证；但完整、通用、可配置的预输入系统还没有完成。
8. 当前已经存在最小敌对攻击来源，并且玩家侧已经具备最小受击结果入口。
9. 基础 Guard 已经正式接入当前 `UE5 + GAS + C++` 基础战斗架构，并完成了一轮白盒验收；它已经不是“待规划能力”，而是“最小可用能力”。
10. 因此，当前第二章基础战斗的真实阶段是：
    **最小攻防闭环已经成立，但完整受击 / 伤害系统与完整格挡规则系统仍未完成。**
11. 当前还没有正式完成的重点包括：
    攻击描述与命中元数据；
    正式伤害结果与最小血量承载；
    正式受击状态与恢复；
    格挡规则二期；
    格挡结果结算；
    预输入二期泛化。
12. 当前不建议再把主线回退到“基础闪避正式落地”或“基础格挡正式实现”；这些都已经完成首轮阶段目标。
13. 当前更合理的下一阶段顺序是：
    攻击描述与命中元数据基础
    -> 玩家伤害结果正式化
    -> 玩家受击状态最小实现
    -> 格挡规则基础升级
    -> 格挡结果结算与反击接口预留
    -> 预输入二期泛化。
14. 项目当前仍在持续把稳定规则回收进 `.trellis/spec/`，并把阶段事实维护在 `.trellis/tasks/05-19-chapter2-basic-combat/` 之下，避免后续 workflow 继续读取过时进度。
