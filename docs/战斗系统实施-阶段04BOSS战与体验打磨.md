# 战斗系统实施-阶段04BOSS战与体验打磨

## 阶段目标

将前面完成的普通攻击、结印、合印、队友能力放入一个完整战斗场景中，通过 BOSS 战验证整个系统。

该阶段完成后，至少要实现：

- 一个可完整进行的 BOSS 战场景
- BOSS 能与玩家和队友形成稳定对抗
- 战斗中可验证普通攻击、技能、合印的实际价值

## 本阶段范围

包含：

- BOSS 最小行为与招式
- 战斗场景验证
- 数值与节奏调试
- 关键反馈优化

不包含：

- 大规模美术演出打磨
- 联机适配
- 最终版数值平衡

## 任务拆分

### 1. 设计 BOSS 最小战斗原型

目标：
先做一个能验证系统的 BOSS，而不是追求完整最终内容。

建议任务：

- 设计基础索敌与攻击
- 至少配置 2 到 3 个招式
- 支持被命中、硬直限制和死亡

完成标准：

- BOSS 可以完成基础战斗循环

### 2. 验证玩家与队友协作价值

目标：
确认合印与队友不是摆设，而是战斗中的有效选择。

建议任务：

- 验证合印增益是否足够明显
- 验证队友在战斗中是否能实际帮助玩家
- 验证玩家单打与协作状态的差异

完成标准：

- 协作机制在战斗中有可感知价值

### 3. 调整战斗反馈

目标：
提升战斗可读性和打击感。

建议任务：

- 优化命中反馈
- 优化受击反馈
- 优化技能释放反馈
- 优化结印/合印状态提示

完成标准：

- 玩家能明显感知战斗状态变化

### 4. 调整战斗节奏与数值

目标：
避免系统虽然能跑，但玩起来杂乱或无聊。

建议任务：

- 调整普通攻击节奏
- 调整技能收益
- 调整 BOSS 攻击频率和空档
- 调整合印收益是否过强或过弱

完成标准：

- 一场战斗具备基本节奏感和可玩性

## 建议产出物

- 一个 BOSS 验证角色
- 一个战斗验证场景
- 一轮数值与节奏调整记录
- 一份后续扩展问题清单

## 验收清单

- BOSS 战可完整进行
- 玩家、队友、BOSS 三方逻辑稳定
- 普攻、技能、合印都能在实战中使用
- 战斗节奏具备基础可玩性

## 后续对话建议切分

1. 先做 BOSS 最小原型
2. 再接入玩家与队友验证
3. 再调反馈
4. 最后调节奏和数值

## 备注

这一阶段的意义是“验证系统整体是否成立”，而不是把所有内容都做到最终版本。
