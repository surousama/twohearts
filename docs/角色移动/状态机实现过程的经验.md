# 状态机实现过程的经验

## Start 状态 —— Distance Matching 搭建踩坑记录

### 最终正确的配置

- **AnimGraph**：Sequence Evaluator（动画：`ChineseWarrior_Run_Start_F_Lfoot`）→ Output Animation Pose
- **Sequence Evaluator 关键设置**：Explicit Time = **Always Dynamic**，Should Loop = false
- **OnUpdate 函数**：通过 Sequence Evaluator 节点的 Details 面板 → Functions → On Update → Create Binding 创建
- **OnUpdate 连线**：
  ```
  OnUpdate Node → Convert to Sequence Evaluator (Succeeded) → Advance Time by Distance Matching → Return Node
  OnUpdate Context → Advance Time by Distance Matching (Update Context)
  Convert Return Value → Advance Time (Sequence Evaluator)
  DistanceTraveled 变量 → Advance Time (Distance Traveled)
  Distance Curve Name = "Distance"
  Play Rate Clamp = (0.75, 1.25)
  Return Node 不连 Return Value
  ```

---

### 踩坑 1：OnUpdate 函数中无法引用 Sequence Evaluator

**现象**：在 OnUpdate 函数中，`Advance Time by Distance Matching` 的 Sequence Evaluator 引脚无法连接 OnUpdate 的 Node 引脚。

**原因**：Node 引脚输出的是 `Anim Node Reference`（通用类型），但 Sequence Evaluator 引脚需要的是 `Sequence Evaluator Reference`（特定类型），类型不匹配。

**解决**：在中间插入 **`Convert to Sequence Evaluator`** 节点（选择有执行线的版本），将 Anim Node Reference 转换为 Sequence Evaluator Reference。

---

### 踩坑 2：OnUpdate 函数的创建方式

**现象**：从左侧 Functions 面板手动添加 OnUpdate 函数后，函数中使用 `Make Sequence Evaluator Reference` 无法找到状态中的 Sequence Evaluator 节点引用。

**原因**：手动创建的函数不会自动绑定到特定的动画节点。

**解决**：必须从 **Sequence Evaluator 节点的 Details 面板** → Functions → On Update → **Create Binding** 创建函数。这样创建的函数会自动带有 Context 和 Node 参数，其中 Node 就是该 Sequence Evaluator 的引用。

---

### 踩坑 3：函数签名不兼容

**现象**：编译报错 `Update function's signature is not compatible. Sequence Evaluator (by time) 'ChineseWarrior_Run_Start_F_Lfoot'`

**原因**：之前残留的 OnUpdate 函数签名（参数类型）与 Sequence Evaluator 期望的不一致。可能是旧的绑定没有完全清除。

**解决**：
1. 先 Remove Binding
2. 删除左侧 Functions 面板中的 OnUpdate 函数
3. Compile 清除错误
4. 重新从 Sequence Evaluator Details 面板 Create Binding

---

### 踩坑 4：Return Node 不能连 Return Value

**现象**：将 `Advance Time by Distance Matching` 的 Return Value 连到 Return Node 后，编译报签名不兼容。

**原因**：OnUpdate 函数不需要返回值。Advance Time 的 Return Value（Sequence Evaluator Reference 类型）不应该作为函数返回值。

**解决**：Return Node **不连任何数据引脚**，只连执行线。

---

### 踩坑 5：动画不播放（卡在第 0 帧）

**现象**：OnUpdate 函数正常调用、Convert to Sequence Evaluator 成功、DistanceTraveled 有值，但动画始终不播放。

**日志警告**：
```
LogAnimDistanceMatchingLibrary: Warning: Could not set explicit time on sequence evaluator, value is not dynamic. Set it as Always Dynamic.
LogAnimation: Warning: Trying to use FAnimCurveBufferAccess on an animation that is not compressed with UniformIndexable
```

**原因**：Sequence Evaluator 的 Explicit Time 默认模式不允许外部动态修改。

**解决**：选中 Sequence Evaluator 节点 → Details 面板 → Settings → **Explicit Time** 旁边的下拉框从 `Pin` 改为 **`Always Dynamic`**。

---

### 踩坑 6：Inertialization 节点缺失

**现象**：运行时报错 `No Inertialization node found for request from transition 'Idle' to 'Start' in state machine 'LocomotionSM'`

**原因**：状态转换线设置了 Inertialization Blend，但 AnimGraph 中状态机输出后面没有 Inertialization 后处理节点。

**解决**：在 AnimGraph 中，LocomotionSM 输出后面插入 **Inertialization** 节点：
```
LocomotionSM → Inertialization → Slot 'DefaultSlot' → Output Pose
```

---

### 经验总结

1. **Sequence Evaluator 的 OnUpdate 绑定必须通过 Details 面板的 Create Binding 创建**，不能手动创建函数
2. **Node 引脚需要 Convert to Sequence Evaluator 转换**后才能连接到 Distance Matching 节点
3. **Explicit Time 必须设为 Always Dynamic**，否则 Distance Matching 无法修改播放时间
4. **OnUpdate 的 Return Node 不连返回值**
5. **使用 Inertialization Blend 的状态机后面必须有 Inertialization 节点**
6. 遇到问题时，先用 Sequence Player 验证状态机流程是否正确，再替换为 Sequence Evaluator + Distance Matching

---

## Stop 状态 —— Distance Match to Target 搭建踩坑记录

### 最终正确的配置

- **AnimGraph**：Sequence Evaluator（动画：`ChineseWarrior_Run_Stop_F_Lfoot`）→ Output Animation Pose
- **Sequence Evaluator 关键设置**：Explicit Time = **Always Dynamic**，Should Loop = false
- **OnUpdate 函数**：通过 Sequence Evaluator 节点的 Details 面板 → Functions → On Update → Create Binding 创建
- **OnUpdate 连线（含分支逻辑）**：
  ```
  OnUpdate Node → Convert to Sequence Evaluator (Succeeded) → Branch (PredictedStopDistance > 1.0)
  
  True 分支（角色还在减速）：
    → Distance Match to Target → Return Node
    Convert Return Value → Distance Match to Target (Sequence Evaluator)
    PredictedStopDistance 变量 → Distance Match to Target (Distance to Target)
    Distance Curve Name = "Distance"
  
  False 分支（角色已停下）：
    → 获取当前时间 + DeltaTimeX → Set Explicit Time → Return Node
    （从当前帧开始按正常速度播放剩余动画）
  ```

### PredictedStopDistance 的计算

使用 `Predict Ground Movement Stop Location` 函数（AnimationLocomotionLibrary 插件）：
```
CharacterMovement → Get Velocity → Predict Ground Movement Stop Location
  参数：Velocity, BrakingFriction, GroundFriction, BrakingFrictionFactor, BrakingDecelerationWalking
  均从 CharacterMovement 组件获取

返回值（偏移向量） → Vector Length XY → Set PredictedStopDistance
注意：返回值是相对偏移量，不是世界坐标！不需要减去 Actor Location！
```

---

### 踩坑 7：Predict Ground Movement Stop Location 返回值误解

**现象**：PredictedStopDistance 始终为 500~1000 的大值，角色停下后也不归零。

**原因**：误以为返回值是世界坐标，用 `返回值 - Actor Location` 计算距离。实际上返回值是**相对于当前位置的偏移向量**。

**解决**：直接对返回值取 `Vector Length XY`，不减 Actor Location。

**教训**：对不确定的函数返回值，先打印中间值验证，不要凭假设编写逻辑。

---

### 踩坑 8：OnUpdate 缺少分支逻辑导致动画卡住

**现象**：角色进入 Stop 状态后卡在一个奇怪的姿势不动。

**原因**：`Distance Match to Target` 在 PredictedStopDistance = 0 时，会把动画定位到 Distance 曲线的起始帧（第 0 帧），导致动画卡在起始姿势。

**解决**：在 OnUpdate 中加入 Branch 分支判断：
- PredictedStopDistance > 1.0 → 执行 Distance Match to Target（距离匹配阶段）
- PredictedStopDistance ≤ 1.0 → 按正常速度播放动画剩余部分（Get Explicit Time + DeltaTimeX → Set Explicit Time）

**教训**：Distance Match to Target 只适用于"角色还在减速"的阶段。角色完全停下后必须切换为正常播放模式。这是 Lyra 的标准做法。

---

### 踩坑 9：Distance Match to Target 的 Sequence Evaluator 引脚未连接

**现象**：OnUpdate 函数被正确调用，但动画不播放。

**原因**：`Distance Match to Target` 的 Sequence Evaluator 引脚漏连了 `Convert to Sequence Evaluator` 的 Return Value。

**教训**：每次搭建新的 OnUpdate 函数时，逐一检查所有数据引脚是否已连接。

---

### Stop 状态经验总结

1. **`Predict Ground Movement Stop Location` 返回的是偏移向量，不是世界坐标**
2. **OnUpdate 必须有分支逻辑**：减速中用 Distance Match，停下后正常播放
3. **Distance 曲线是递增的**（从 0 到最大位移距离），代表动画播放过程中根骨骼的累计位移
4. **`Distance Match to Target` 在距离为 0 时会跳到第 0 帧**，所以必须在距离接近 0 时切换播放模式
5. Start 状态的通用经验（Always Dynamic、Create Binding、Convert 节点、Return Node 不连返回值）同样适用于 Stop 状态

## 手感调试经验

### Sequence Evaluator 重入时需要重置 Explicit Time

**现象**：第一次进入 Start 状态正常，第二次进入时动画瞬间播完。

**原因**：Sequence Evaluator 的 Explicit Time 保留了上次的值（动画末尾），重新进入时没有重置为 0。

**解决**：为 Sequence Evaluator 添加 **On Become Relevant** 绑定函数，在函数中 `Convert to Sequence Evaluator` → `Set Explicit Time (0.0)`。Start 和 Stop 都需要设置。

---

### 起步加速度调整

**现象**：第二次起步时 DistanceTraveled 一开始就很大（8+），导致 Start 动画被 Distance Matching 快速推进，起步动画几乎看不到。

**原因**：UE5 默认 Max Acceleration = 2048，角色起步瞬间就有很高的速度。

**解决**：将 Character Movement 的 **Max Acceleration** 从 2048 降为 **1300**，起步加速变缓，Distance Matching 有足够的帧来逐步推进动画。

---

### 刹车距离调整

**现象**：角色松手后瞬间停下，像"钉在地上"。

**原因**：默认参数下实际减速力 = BrakingDecelerationWalking + Speed × GroundFriction × BrakingFrictionFactor，值非常大，刹车时间只有 0.07 秒。

**解决**：调整以下参数实现自然的减速滑行：

| 参数 | 默认值 | 调整后 |
|------|--------|--------|
| Braking Deceleration Walking | 2000 | 500 |
| Ground Friction | 8 | 4 |
| Braking Friction Factor | 2 | 0.5 |
| Max Acceleration | 2048 | 1300 |

刹车公式：减速力 = BrakingDecelerationWalking + Speed × GroundFriction × BrakingFrictionFactor
调整后在 Max Walk Speed = 500 时：减速力 ≈ 1500，刹车时间 ≈ 0.33 秒，刹车距离 ≈ 83cm。

---

### Stop → Idle 转换条件

**现象**：Stop 状态持续 3.4 秒才进入 Idle，明显过长。

**原因**：Distance Match 只推进到 0.95 秒就结束，NormalPlay 从 0.95 秒按正常速度播放剩余部分，而 Stop 动画有效内容到 2.5 秒（75帧），后面还有 1.57 秒空白帧。

**解决**：Stop → Idle 的转换条件改为基于速度判断 `GroundSpeed < 5.0`，不依赖动画播放进度。Blend Time 设为 0.3 秒。这样角色物理停下后立即过渡到 Idle。

---

### Sync Group 设置

**现象**：Start → Cycle 过渡时脚步打结。

**原因**：两个状态的动画没有加入同步组，过渡时脚步相位不匹配。

**解决**：所有 Locomotion 状态机中的动画节点都设置：
- Group Name = `Locomotion`
- Group Role = `Can be Leader`
- Method = **`SyncGroup`**（不是 Do Not Sync）

---

### Stop → Cycle 转换线

**现象**：在 Stop 状态中按移动键，角色产生严重滑步。

**原因**：没有 Stop → Cycle 的转换线，角色只能等 Stop 播完 → Idle → Start，期间已经在移动但还在播 Stop 动画。

**解决**：添加 Stop → Cycle 转换线，条件 `bShouldMove == true`，Blend Time = 0.15。
