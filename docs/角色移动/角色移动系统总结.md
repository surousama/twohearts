# 角色移动系统总结

## 1. Event Graph 核心变量

| 变量名 | 含义 |
|------|------|
| `GroundSpeed` | 水平移动速度 |
| `DistanceTraveled` | 上一帧移动距离，通常为 `GroundSpeed × DeltaTime` |
| `bShouldMove` | 是否应进入移动状态 |
| `MovementDirection` | 移动方向角，范围 `-180 ~ 180` |
| `Velocity` | 速度向量缓存 |
| `Acceleration` | 加速度向量缓存 |
| `DeltaTimeX` | 帧间隔时间缓存 |
| `CharacterMovement` | 运动组件引用缓存 |
| `PredictedStopDistance` | 预测停步剩余距离 |
| `bShouldPivot` | 是否应进入 Pivot |
| `PredictedPivotDistance` | 预测折返点距离 |
| `bRightFootForward` | 由 `FootPosition` 曲线判断，用于选脚 |

## 2. 状态实现要点

### 2.1 Stop

- 使用 `Sequence Evaluator`
- 进入状态时先按方向选择 Stop 动画
- `OnUpdate` 分两段处理：
  - 仍在减速时：使用 `Distance Match to Target`
  - 低速或接近停下时：放弃距离匹配，切回普通时间推进
- `Stop -> Idle` 建议按速度退出，不依赖动画是否完整播完
- `Stop -> Cycle` 转换线必须存在，否则会出现移动中继续播 Stop 的滑步问题

### 2.2 Pivot

- 触发条件核心判断为速度方向与加速度方向相反，并且速度高于阈值
- `OnUpdate` 一般分为两段：
  - 急停阶段：使用 `Distance Match to Target`
  - 反向起步阶段：使用 `Advance Time by Distance Matching`

## 3. 选脚方案

- 当前保留的方案：基于 `FootPosition` 曲线显式选脚
- `FootPosition` 的正确语义：表示“即将迈出的脚”，不是“刚着地的脚”
- 曲线在双脚交替阶段必须平滑经过 `0`
- 运行时可直接判断：
  - `FootPosition > 0`：选 `Rfoot`
  - `FootPosition < 0`：选 `Lfoot`

## 4. 踩坑经验浓缩

1. `Distance Match to Target` 不适合在角色已经基本停下时继续使用，否则容易卡姿势或跳回错误时间点
2. `Predict Ground Movement Stop Location` 如果被误当成世界坐标使用，StopDistance 会整体失真
3. `Rewind Debugger` 适合排查状态切换、选脚、动画引用是否选错，比简单打印更有效
