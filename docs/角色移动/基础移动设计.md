# 基础战斗系统设计

## 1. 角色的移动控制

### 1.1 绝区零移动系统分析

基于对绝区零游戏的分析，其角色移动控制系统包含以下核心要素：

#### 1.1.1 移动状态层次

绝区零的角色移动系统采用了分层状态设计：

**基础移动状态**：

- **静止状态（Idle）**：角色站立待机，播放呼吸动画
- **起步状态（Start Moving）**：从静止到移动的过渡动画，衔接Idle与Walk
- **行走状态（Walk）**：低速移动，适用于精确定位，是移动系统的核心枢纽状态
- **跑步状态（Run）**：中速移动，日常移动速度
- **疾跑状态（Sprint）**：高速移动，持续跑步一段时间后自动进入
- **折返状态（Pivot）**：角色在移动中进行方向折返的过渡状态，表现角色急转身的动作
- **停步状态（Stop Moving）**：从移动到静止的过渡动画

**特殊移动状态**：

- **闪避状态（Dodge）**：快速位移，具有无敌帧
- **冲刺攻击（Dash Attack）**：闪避后的攻击派生
- **精准闪避（Perfect Dodge）**：时机完美的闪避，触发"魔女时间"

#### 1.1.2 移动状态转换关系

```
                         Idle
                          ↓
                     StartMoving ──→ Stop Moving
                          ↓               ↑
            Pivot ←→  Walk  ──────────────┤
                       ↕                  │
                      Run  ───────────────┤
                       ↕                  │
                     Sprint ──────────────┘
                                          ↓
                                        Idle
```

**具体转换路径说明**：

- **Idle → StartMoving**：玩家输入移动指令，从静止进入起步过渡
- **StartMoving → Walk**：起步动画完成，进入行走状态
- **StartMoving → Stop Moving**：起步过程中释放移动输入，直接进入停步
- **Walk ↔ Pivot**：行走中输入反方向时进入折返，折返完成后回到行走
- **Walk ↔ Run**：行走持续加速进入跑步，跑步减速回到行走
- **Run ↔ Sprint**：跑步持续加速进入疾跑，疾跑减速回到跑步
- **Walk → Stop Moving**：行走状态下释放移动输入，进入停步
- **Run → Stop Moving**：跑步状态下释放移动输入，进入停步
- **Sprint → Stop Moving**：疾跑状态下释放移动输入，进入停步
- **Stop Moving → Idle**：停步动画播放完毕，回到静止状态


*注：本章节基于绝区零移动系统的分析制定，具体实现细节将在后续讨论中逐步确认。*
