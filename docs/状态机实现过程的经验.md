# 状态机实现过程的经验

## Start 状态 —— Distance Matching 搭建踩坑记录

### 最终正确的配置

- **AnimGraph**：Sequence Evaluator（动画：`ChineseWarrior_Run_Start_F_Lfoot`）→ Output Animation Pose
- **Sequence Evaluator 关键设置**：Explicit Time = **Always Dynamic**，Should Loop = false
- **OnUpdate 函数**：通过 Sequence Evaluator 节点的 Details 面板 → Functions → On Update → Create Binding 创建
- **OnUpdate 连线**：
  ```
  OnUpdate Node → Convert to Sequence Evaluator (Succeeded) → Advance Time by Distance Matching → Return Node
  OnUpdate Context → Advance Time by Distance Matching (Update Context)
  Convert Return Value → Advance Time (Sequence Evaluator)
  DistanceTraveled 变量 → Advance Time (Distance Traveled)
  Distance Curve Name = "Distance"
  Play Rate Clamp = (0.75, 1.25)
  Return Node 不连 Return Value
  ```

---

### 踩坑 1：OnUpdate 函数中无法引用 Sequence Evaluator

**现象**：在 OnUpdate 函数中，`Advance Time by Distance Matching` 的 Sequence Evaluator 引脚无法连接 OnUpdate 的 Node 引脚。

**原因**：Node 引脚输出的是 `Anim Node Reference`（通用类型），但 Sequence Evaluator 引脚需要的是 `Sequence Evaluator Reference`（特定类型），类型不匹配。

**解决**：在中间插入 **`Convert to Sequence Evaluator`** 节点（选择有执行线的版本），将 Anim Node Reference 转换为 Sequence Evaluator Reference。

---

### 踩坑 2：OnUpdate 函数的创建方式

**现象**：从左侧 Functions 面板手动添加 OnUpdate 函数后，函数中使用 `Make Sequence Evaluator Reference` 无法找到状态中的 Sequence Evaluator 节点引用。

**原因**：手动创建的函数不会自动绑定到特定的动画节点。

**解决**：必须从 **Sequence Evaluator 节点的 Details 面板** → Functions → On Update → **Create Binding** 创建函数。这样创建的函数会自动带有 Context 和 Node 参数，其中 Node 就是该 Sequence Evaluator 的引用。

---

### 踩坑 3：函数签名不兼容

**现象**：编译报错 `Update function's signature is not compatible. Sequence Evaluator (by time) 'ChineseWarrior_Run_Start_F_Lfoot'`

**原因**：之前残留的 OnUpdate 函数签名（参数类型）与 Sequence Evaluator 期望的不一致。可能是旧的绑定没有完全清除。

**解决**：
1. 先 Remove Binding
2. 删除左侧 Functions 面板中的 OnUpdate 函数
3. Compile 清除错误
4. 重新从 Sequence Evaluator Details 面板 Create Binding

---

### 踩坑 4：Return Node 不能连 Return Value

**现象**：将 `Advance Time by Distance Matching` 的 Return Value 连到 Return Node 后，编译报签名不兼容。

**原因**：OnUpdate 函数不需要返回值。Advance Time 的 Return Value（Sequence Evaluator Reference 类型）不应该作为函数返回值。

**解决**：Return Node **不连任何数据引脚**，只连执行线。

---

### 踩坑 5：动画不播放（卡在第 0 帧）

**现象**：OnUpdate 函数正常调用、Convert to Sequence Evaluator 成功、DistanceTraveled 有值，但动画始终不播放。

**日志警告**：
```
LogAnimDistanceMatchingLibrary: Warning: Could not set explicit time on sequence evaluator, value is not dynamic. Set it as Always Dynamic.
LogAnimation: Warning: Trying to use FAnimCurveBufferAccess on an animation that is not compressed with UniformIndexable
```

**原因**：Sequence Evaluator 的 Explicit Time 默认模式不允许外部动态修改。

**解决**：选中 Sequence Evaluator 节点 → Details 面板 → Settings → **Explicit Time** 旁边的下拉框从 `Pin` 改为 **`Always Dynamic`**。

---

### 踩坑 6：Inertialization 节点缺失

**现象**：运行时报错 `No Inertialization node found for request from transition 'Idle' to 'Start' in state machine 'LocomotionSM'`

**原因**：状态转换线设置了 Inertialization Blend，但 AnimGraph 中状态机输出后面没有 Inertialization 后处理节点。

**解决**：在 AnimGraph 中，LocomotionSM 输出后面插入 **Inertialization** 节点：
```
LocomotionSM → Inertialization → Slot 'DefaultSlot' → Output Pose
```

---

### 经验总结

1. **Sequence Evaluator 的 OnUpdate 绑定必须通过 Details 面板的 Create Binding 创建**，不能手动创建函数
2. **Node 引脚需要 Convert to Sequence Evaluator 转换**后才能连接到 Distance Matching 节点
3. **Explicit Time 必须设为 Always Dynamic**，否则 Distance Matching 无法修改播放时间
4. **OnUpdate 的 Return Node 不连返回值**
5. **使用 Inertialization Blend 的状态机后面必须有 Inertialization 节点**
6. 遇到问题时，先用 Sequence Player 验证状态机流程是否正确，再替换为 Sequence Evaluator + Distance Matching
