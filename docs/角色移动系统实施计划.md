```
Event Blueprint Update Animation (DeltaTimeX)
  → Cast To Character → Set DeltaTimeX → Set CharacterMovement

```

#### 通用方向选择函数 FSelectDirectionalAnim
- 输入：Direction (Float) + 8 个方向的动画引用
- 逻辑：`(Direction + 202.5) / 45 → Floor → % 8 → Select`
- 映射：Index 0=B, 1=BL, 2=LL, 3=FL, 4=F, 5=FR, 6=RR, 7=BR
- 标记为**线程安全**（AnimBP 绑定函数要求）
- Start / Stop / Pivot 的 OnBecomeRelevant 中复用

#### 调试方式
- 运行时按 \` 键打开控制台，输入 `ShowDebug Animation` 查看完整动画调试信息

#### Pivot 状态实现（基于 Lyra 实际做法）

**原理**：处理角色移动中突然反方向输入的回转动画（急停 → 转身 → 反向起步）

**触发条件**：
```
(Velocity2D · Acceleration2D) < 0  AND  GroundSpeed > 50
```

**已完成的步骤**：
- [x] Event Graph 新增 bShouldPivot 变量（点积计算）
- [x] Event Graph 新增 PredictedPivotDistance 变量（PredictGroundMovementPivotLocation）
- [x] 添加 Cycle → Pivot 转换线（条件：bShouldPivot == true）
- [x] Pivot → Cycle 转换线（自动规则）
- [x] Pivot 内部用 Sequence Player 跑通流程验证
- [x] 升级为 Sequence Evaluator + Distance Matching 两阶段驱动
- [x] 创建 CC_UniformIndexable（Curve Compression Settings）解决 Distance 曲线压缩问题
- [x] 修复 Branch 条件：从 GroundSpeed > 30 改为 bShouldPivot（速度与加速度是否仍然相反）

**Event Graph 新增变量**：

| 变量名 | 类型 | Category | 说明 |
|--------|------|----------|------|
| bShouldPivot | Bool | Movement | 速度与加速度点积 < 0 且 GroundSpeed > 50 |
| PredictedPivotDistance | Float | Movement | PredictGroundMovementPivotLocation 返回值的 Vector Length |

**OnUpdate 连线方案（Lyra 做法）**：
```
Node → Convert to Sequence Evaluator (Succeeded) → Branch (bShouldPivot)

True（急停阶段，速度与加速度仍然相反，还在靠近回转点）：
  → Distance Match to Target
    - Distance to Target: PredictedPivotDistance
    - Distance Curve Name: "Distance"
  → Return Node

False（反向起步阶段，速度与加速度同向，已过回转点开始反向加速）：
  → Advance Time by Distance Matching
    - Distance Traveled: DistanceTraveled
    - Distance Curve Name: "Distance"
  → Return Node
```

**踩坑记录**：
1. Distance 曲线压缩问题：所有使用 Distance Matching 的动画必须设置 Curve Compression Settings 为 `CC_UniformIndexable`（Codec: UniformIndexable），否则 `FAnimCurveBufferAccess` 报错且打包后失效
2. Branch 条件不能用 GroundSpeed > 30，因为到达回转点时角色可能仍有惯性速度，导致一直走 Distance Match 分支而 PredictedPivotDistance 已经为 0，动画卡死在第一帧
3. 正确做法：用 bShouldPivot（速度·加速度点积）判断，到达回转点后速度与加速度变为同向，自动切换到 Advance Time 推进动画

**待修复——Pivot 朝向问题**：
- [ ] 角色启用了 Orient Rotation to Movement，Pivot 时角色先转向了新方向，然后才播放 Pivot 动作，导致动画方向完全反了
- [ ] 解决方案：Pivot On Become Relevant 中临时禁用 Orient Rotation to Movement，Pivot 结束后恢复
  ```
  On Become Relevant:
    → CharacterMovement → Set Orient Rotation to Movement = false
  
  On Left（或 Cycle/Stop 的 On Become Relevant）:
    → CharacterMovement → Set Orient Rotation to Movement = true
  ```
