### Event Graph（计算层）—— ✅ 已完成

变量列表：

| 变量名 | 类型 | Category | 说明 |
|--------|------|----------|------|
| GroundSpeed | Float | Movement | 水平移动速度（Vector Length XY） |
| DistanceTraveled | Float | Movement | 上一帧移动距离（GroundSpeed × DeltaTime） |
| bShouldMove | Bool | Movement | GroundSpeed > 3.0 AND Acceleration Length > 0 |
| MovementDirection | Float | Movement | 移动方向角度，-180~180（Calculate Direction） |
| PredictedStopDistance | Float | Movement | 预测刹车距离（Speed² / (2 × BrakingDecelerationWalking)） |
| Velocity | Vector | Movement | 缓存角色速度向量 |
| Acceleration | Vector | Movement | 缓存角色加速度向量 |
| DeltaTimeX | Float | Movement | 缓存帧间隔时间 |
| CharacterMovement | CharacterMovementComponent | Movement | 缓存运动组件引用 |

蓝图连线流程：
```
Event Blueprint Update Animation (DeltaTimeX)
  → Cast To Character → Set DeltaTimeX → Set CharacterMovement
  → Set Velocity → Set Acceleration → Set GroundSpeed → Set bShouldMove
  → Set MovementDirection → Set DistanceTraveled → Set PredictedStopDistance
```

#### Pivot 状态实现计划

**原理**：处理角色移动中突然反方向输入的回转动画（急停 → 转身 → 反向起步）

**触发条件**：
```
(Velocity2D · Acceleration2D) < 0  AND  GroundSpeed > 50
```
即当前速度方向和加速度方向相反（玩家做反向输入），且角色有一定速度。

**需要新增的变量**：

| 变量名 | 类型 | 说明 |
|--------|------|------|
| bShouldPivot | Bool | 速度与加速度点积 < 0 且 GroundSpeed > 50 |
| CardinalDirectionFromAcceleration | Enum/Float | 加速度方向（用于选择对应方向的 Pivot 动画） |
| PivotStartDirection | Float | 进入 Pivot 时记录的初始方向 |
| IsMovingPerpendicularToInitialPivot | Bool | 当前移动是否垂直于初始 Pivot 方向 |

**Pivot 内部子状态机 PivotSM**：
- **Pivot A**：第一次回转
- **Pivot B**：连续回转（加速度方向再次变化时切换）
- 两者之间切换条件：加速度方向和上一帧不同

**每个子状态的 Distance Matching**：
1. 阶段一（急停）：`Distance Match to Target`（类似 Stop）+ 分支判断
2. 阶段二（反向起步）：`Advance Time by Distance Matching`（类似 Start）

**状态转换**：

| 转换 | 条件 |
|------|------|
| Cycle → Pivot | bShouldPivot == true |
| Start → Pivot | bShouldPivot == true |
| Pivot → Cycle | 动画播完，或 IsMovingPerpendicularToInitialPivot == true |
| Pivot → Stop | bShouldMove == false |

**配套后处理**（Pivot 过程中需要）：
- Orientation Warping：旋转下半身朝向新方向
- Stride Warping：调整步幅匹配实际速度

**简化实现步骤**：
1. Event Graph 新增 bShouldPivot 变量（点积计算）
2. 添加 Cycle → Pivot、Pivot → Cycle、Pivot → Stop 转换线
3. Pivot 内部先用 Sequence Player 跑通流程
4. 升级为 Sequence Evaluator + Distance Matching（急停 + 反向起步两阶段）
5. 加入 Pivot A/B 子状态机处理连续反向
6. 加入垂直移动检测优化退出条件
