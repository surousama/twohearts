# 基础战斗系统设计

## 1. 角色的移动控制

### 1.1 绝区零移动系统分析

基于对绝区零游戏的分析，其角色移动控制系统包含以下核心要素：

#### 1.1.1 移动状态层次
绝区零的角色移动系统采用了分层状态设计：

**基础移动状态**：
- **静止状态（Idle）**：角色站立待机，播放呼吸动画
- **起步状态（Start Moving）**：从静止到移动的过渡动画，衔接Idle与Walking
- **行走状态（Walking）**：低速移动，适用于精确定位，是移动系统的核心枢纽状态
- **疾走状态（Jogging）**：中速移动，日常移动速度
- **疾跑状态（Running）**：高速移动，持续疾走一段时间后自动进入
- **折返状态（Pivot）**：角色在移动中进行方向折返的过渡状态，表现角色急转身的动作
- **停步状态（Stop Moving）**：从移动到静止的过渡动画

**特殊移动状态**：
- **闪避状态（Dodge）**：快速位移，具有无敌帧
- **冲刺攻击（Dash Attack）**：闪避后的攻击派生
- **精准闪避（Perfect Dodge）**：时机完美的闪避，触发"魔女时间"

#### 1.1.2 移动状态转换关系

```
                         Idle
                          ↓
                     StartMoving ──→ Stop Moving
                          ↓               ↑
            Pivot ←→ Walking ─────────────┤
                       ↕                  │
                    Jogging ──────────────┤
                       ↕                  │
                    Running ──────────────┘
                                          ↓
                                        Idle
```

**具体转换路径说明**：
- **Idle → StartMoving**：玩家输入移动指令，从静止进入起步过渡
- **StartMoving → Walking**：起步动画完成，进入行走状态
- **StartMoving → Stop Moving**：起步过程中释放移动输入，直接进入停步
- **Walking ↔ Pivot**：行走中输入反方向时进入折返，折返完成后回到行走
- **Walking ↔ Jogging**：行走持续加速进入疾走，疾走减速回到行走
- **Jogging ↔ Running**：疾走持续加速进入疾跑，疾跑减速回到疾走
- **Walking → Stop Moving**：行走状态下释放移动输入，进入停步
- **Jogging → Stop Moving**：疾走状态下释放移动输入，进入停步
- **Running → Stop Moving**：疾跑状态下释放移动输入，进入停步
- **Stop Moving → Idle**：停步动画播放完毕，回到静止状态

#### 1.1.3 移动控制特点

**输入响应机制**：
- **8方向移动**：键盘控制支持8个基本方向
- **360度移动**：手柄/触屏支持更灵活的移动方向
- **渐进加速**：移动速度从疾走逐渐提升到疾跑
- **视觉反馈**：疾跑状态有白雾特效和动画变化

**闪避系统特色**：
- **无次数限制**：基础闪避不受使用次数限制
- **连续使用惩罚**：短时间内连续闪避会进入1秒冷却
- **方向控制**：可指定闪避方向，默认向后闪避
- **派生动作**：闪避后可连接冲刺攻击

**摄像机联动**：
- **跟随角色**：摄像机绑定在角色身上
- **方向调整**：左右移动时摄像机会轻微跟随调整
- **面朝方向**：默认朝向角色面对的方向

### 1.2 双心印项目移动系统设计目标

基于绝区零的优秀设计，结合双心印项目的战斗需求，我们的移动系统应该实现：

#### 1.2.1 核心设计原则
- **流畅性**：状态转换自然，无明显卡顿
- **响应性**：输入反馈及时，操作手感良好
- **战术性**：移动与战斗紧密结合，支持战术操作
- **表现力**：丰富的动画表现，提升视觉体验

#### 1.2.2 功能需求
- **基础移动**：支持多方向移动和速度变化
- **闪避系统**：灵活的闪避机制，支持战斗中的位置调整
- **结印移动**：结印过程中的特殊移动状态
- **合印协作**：双人合印时的位置协调机制

#### 1.2.3 技术实现要点
- **状态机管理**：清晰的移动状态转换逻辑
- **动画融合**：平滑的动画过渡和混合
- **物理集成**：与UE5物理系统的良好集成
- **网络同步**：为未来多人功能预留同步机制

### 1.3 待确认的设计细节

以下设计细节需要进一步讨论和确认：

1. **移动速度数值**：各状态的具体移动速度
2. **加速曲线**：从疾走到疾跑的加速过程
3. **闪避参数**：闪避距离、无敌帧时长、冷却时间
4. **动画资源**：需要制作的动画类型和数量
5. **特效表现**：移动状态的视觉特效设计
6. **音效配合**：移动动作的音效反馈
